AutoLaunch Cloud Pipeline

A containerized build server that clones a frontend Git repository, installs dependencies, runs a production build, and uploads static assets to Amazon S3 — similar to the core build-and-deploy flow used by platforms like Vercel and Netlify.

View on GitHub

What It Does

  • Builds frontend apps in a container for consistent, reproducible builds.
  • Deploys static assets to S3 so websites can be served from cloud storage/CDNs.
  • Stores outputs under a unique project ID to support multiple deployments.

How It Works

  1. Accepts a Git repository URL and a project ID.
  2. Clones the repo inside a Docker container.
  3. Runs npm install and npm run build.
  4. Uploads files from dist/ to Amazon S3.
  5. Stores outputs under s3://<bucket>/<project-id>/.

Run with Docker

Build and run the containerized build server using the commands below:

# Build the Docker image
docker build -t build-server .

# Run the Docker container
docker run --rm \
  -e GIT_REPOSITORY_URL=https://github.com/user/repo.git \
  -e PROJECT_ID=my-project \
  -e AWS_REGION=us-east-1 \
  -e AWS_ACCESS_KEY_ID=your-access-key-id \
  -e AWS_SECRET_ACCESS_KEY=your-secret-access-key \
  build-server

Replace the placeholder values with your Git repository URL, AWS credentials, and project ID. For production use, prefer IAM roles or scoped credentials instead of long-lived keys.

Why This Project Matters

This project demonstrates practical DevOps fundamentals: containerized build reproducibility, CI/CD-style automation, artifact storage, and cloud deployment patterns. It also highlights how modern “deploy platforms” work under the hood.
