DevOps Copilot

A natural language DevOps agent that helps engineers manage cloud infrastructure using plain English. DevOps Copilot interprets requests, maintains conversational context, and executes safe operations across Kubernetes and AWS — directly from a Python-based CLI.

View on GitHub

Why I Built This

DevOps workflows are powerful but fragmented — engineers jump between kubectl, Helm, AWS tooling, YAML manifests, and scripts just to answer simple questions or complete routine tasks. DevOps Copilot reduces that friction by acting as a conversational layer over real infrastructure tools, while still keeping humans in control.

What It Enables

Instead of memorizing commands, you can describe what you want:

  • “List all running pods in the default namespace.”
  • “Launch three EC2 instances in the test environment.”
  • “Delete the bucket I just created.”

The agent converts these into structured actions and executes them using Kubernetes and AWS integrations, with safeguards for any destructive or high-risk operations.

Core Capabilities

  • Conversational Intelligence: Uses an LLM via LangChain to interpret intent and map it to DevOps actions.
  • Context Memory: Tracks recent entities so phrases like “that deployment” resolve correctly.
  • Approval Gate: Requires explicit confirmation before risky commands (delete/terminate/overwrite).
  • Reasoning Visibility: Streams intermediate reasoning to the console in a subtle, readable format.
  • Auditability: Logs actions so you can trace what was executed and when.

Integrations

Kubernetes

  • Resource listing, describe, apply, delete, and watch
  • Helm chart management
  • TTL-based resource caching to reduce repeated API calls

AWS

  • EC2 instance lifecycle (launch, terminate, modify)
  • S3 operations (create, upload, list, delete)
  • DynamoDB tables and item management
  • Retries + pagination handling for reliability

Docker (In Progress)

  • Image build workflows, container lifecycle management, registry operations

Getting Started

Clone the repo, configure credentials, and start the REPL. The agent will use your existing AWS / Kubernetes setup (profiles, role assumption, kubeconfig) the same way you already do locally. 

git clone https://github.com/anchalthool/devopscopilot.git
cd devopscopilot

# Create a .env file with:
OPENAI_API_KEY=...
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_REGION=us-east-1
MODEL_NAME=gpt-4
TEMPERATURE=0.3
