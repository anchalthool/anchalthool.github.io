Disappearing Encrypted Group Messaging

A privacy-first, WhatsApp-style group chat web app with time-bound groups, secure invite onboarding, and browser-based end-to-end encryption. Groups automatically expire and revoke access, enabling lifecycle-managed collaboration for sensitive or temporary conversations.

View on GitHub

Why I Built This

Many group chats are effectively permanent — messages live forever, members linger, and old links keep working. I wanted to explore a model that feels more “ephemeral”: groups should have a clear lifecycle, invites should expire, and the server should never see plaintext messages. This project was built to learn end-to-end encryption workflows, privacy-first design, and secure group lifecycle enforcement.

What It Does

  • End-to-end encrypted group chat: Messages are encrypted in the browser and only decrypted by members.
  • Disappearing groups: Groups automatically expire (1 day / 7 days / custom date-time).
  • Secure invite onboarding: Expiring invite links allow controlled access via tokens.
  • Role-based access: Group creators act as admins and control invite creation.
  • WhatsApp-inspired UI: Two-column layout with chat bubbles and expiry indicators.

Security Model

The backend stores only ciphertext — it never receives or persists plaintext messages. Encryption and decryption happen entirely in the client using the browser’s Web Crypto APIs. Group keys are derived from a passphrase, so only users with the passphrase can decrypt the chat history.

Architecture Overview

Layer Implementation
Frontend React (Vite), Web Crypto API (AES-GCM encryption/decryption)
Backend Node.js + Express, JWT auth, lifecycle rules
Database PostgreSQL (groups, memberships, invites, encrypted messages)

Core UX Flows

  • Create group: choose expiry policy (1 day / 7 days / custom time)
  • Invite members: admin generates token-based invite links that can expire
  • Join group: user logs in → auto-joins via token → enters passphrase → decrypts messages
  • Expiry: once expired, messaging and access are revoked automatically

Run Locally

Backend

cd server
npm install
npm run dev

Create server/.env:

PORT=4000
DATABASE_URL=postgres://user:password@localhost:5432/disappearing_groups
JWT_SECRET=your-secret-key

Frontend

cd client
npm install
npm run dev

Invite Flow

Open the invite URL:

http://localhost:<vite_port>/join/<token>

Login → auto-join → enter passphrase → decrypt messages.

What This Project Shows

This project demonstrates full-stack security-aware engineering: client-side encryption, key derivation, token-based onboarding, group lifecycle enforcement, and clean UX patterns for privacy-first products. It’s a practical exploration of “zero-knowledge” storage applied to real-time group collaboration.
